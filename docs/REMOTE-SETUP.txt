   1) Nhận TRIGGER alert (treo/dừng) + xem ĐẦY ĐỦ training log.
   2) NHẮN cho Claude (agent) để tự kill / quan sát / ghi nhật ký / chạy lại.
  Cập nhật: 2026-05-30
--------------------------------------------------------------------------------
  KIẾN TRÚC TỔNG (mọi thứ chạy trên DESKTOP có 4090):
    DESKTOP (Windows + WSL2)
      ├─ tmux session "ml": [0] training  [1] Claude Code (agent)  [2] monitor_train.py
      ├─ (truy cập từ xa: Chrome Remote Desktop HOẶC Tailscale+SSH)
    PHONE/LAPTOP:
      ├─ Telegram app   → nhận TRIGGER + log on-demand (đường "biết")
      └─ Remote access  → attach tmux → NHẮN Claude + xem full log (đường "làm")

================================================================================
  PHẦN B — TMUX  (giữ session BỀN + chia sẻ — BẮT BUỘC cho cả 2 đường)
================================================================================
  tmux giữ training + Claude SỐNG kể cả khi bạn ngắt kết nối, và cho phép attach lại
  từ phone/laptop để xem + gõ.

  Cài + tạo session (trong WSL trên desktop):
    sudo apt install -y tmux
    tmux new -s ml                      # tạo session "ml"
    # trong tmux: Ctrl-b c = tạo window mới; Ctrl-b <số> = chuyển; Ctrl-b d = detach
    #   window 0: chạy training       → python train.py --episodes 4000
    #   window 1: chạy Claude Code     → claude        (đây là chỗ bạn NHẮN tôi)
    #   window 2: chạy monitor         → python monitor_train.py
    #   window 3 (tùy): tail -f results/result_*/training_log.txt   (full log live)

  Attach lại (từ desktop / qua SSH):  tmux attach -t ml
  → training & Claude vẫn chạy dù bạn đóng SSH/đóng app. Chỉ mất khi desktop tắt/reboot.


================================================================================
  PHẦN C — TRUY CẬP TỪ XA  (chọn 1 trong 2 tier)
================================================================================

── TIER 1 (DỄ NHẤT, khuyến nghị): CHROME REMOTE DESKTOP ───────────────────────
  Xem TOÀN BỘ desktop (terminal training + Claude + log) từ phone/laptop, gõ trực tiếp.
  1. Desktop: cài "Chrome Remote Desktop" (remotedesktop.google.com/access) → đặt PIN.
  2. Phone: app "Chrome Remote Desktop" (iOS/Android), login cùng Google account → chọn
     desktop → nhập PIN → thấy màn hình desktop.
  3. Laptop: vào remotedesktop.google.com/access (trình duyệt) → kết nối.
  → Mở cửa sổ WSL/terminal đang chạy tmux → `tmux attach -t ml` → chuyển window 1 →
    GÕ tin nhắn cho Claude (kill/observe/rerun). Window 3 xem full log.
  Ưu: 5 phút, không cấu hình mạng, phone+laptop đều được. Nhược: nặng hơn (chia sẻ màn hình).

── TIER 2 (NHẸ, terminal-clean): TAILSCALE + SSH ──────────────────────────────
  Chỉ terminal (ít data, hợp phone mạng yếu). Nhiều bước hơn (mạng WSL hơi fiddly).
  1. Tailscale (VPN mesh, free, không port-forward):
     - Desktop (Windows): cài Tailscale, login.
     - Phone + laptop: cài Tailscale, login CÙNG account → mọi máy thấy nhau qua IP riêng.
  2. SSH server trong WSL:
     sudo apt install -y openssh-server
     sudo sed -i 's/#Port 22/Port 2222/' /etc/ssh/sshd_config   # port riêng cho WSL
     sudo service ssh start            # (hoặc: sudo /usr/sbin/sshd)
  3. Cho Tailscale tới được WSL: cách đơn giản nhất = chạy Tailscale TRONG WSL:
     curl -fsSL https://tailscale.com/install.sh | sh
     sudo tailscaled --tun=userspace-networking >/tmp/ts.log 2>&1 &
     sudo tailscale up                 # login → WSL có IP Tailscale riêng
     (Win11 có systemd: thêm '[boot] systemd=true' vào /etc/wsl.conf rồi
      'sudo systemctl enable --now tailscaled' cho gọn.)
  4. Từ laptop:  ssh -p 2222 <user>@<wsl-tailscale-ip>  → tmux attach -t ml
     Từ phone:   app Termius/Blink → SSH tới <wsl-tailscale-ip>:2222 → tmux attach -t ml
  → Trong tmux: window 1 = NHẮN Claude; `tail -f .../training_log.txt` = full log.
  Ưu: nhẹ, bảo mật, hợp phone. Nhược: cấu hình mạng WSL hơi rối (nếu kẹt → dùng Tier 1).


================================================================================
  PHẦN D — WORKFLOW HÀNG NGÀY
================================================================================
  Khởi động (1 lần, trên desktop):
    1. tmux new -s ml
    2. window 0: python train.py --episodes 4000
    3. window 1: claude   (mở agent — phiên làm việc với tôi)
    4. window 2: python monitor_train.py   (Telegram alerts)
    5. Ctrl-b d (detach) — mọi thứ chạy nền.

  Từ phone/laptop khi nhận alert (vd ⚠ TREO từ Telegram):
    1. Mở Chrome Remote Desktop (Tier1) HOẶC SSH+tmux attach (Tier2).
    2. Vào window 1 (Claude) → nhắn: "check result_N" / "training treo, kill rồi rerun" /
       "ghi nhật ký" → tôi thực hiện (kill, quan sát, cập nhật nhật ký, chạy lại) y như
       khi bạn ngồi máy.
    3. Window 3 / `tail -f` → xem full log realtime.

  → Telegram = NHẬN BIẾT (push, không cần mở gì). Remote+tmux = LÀM (nhắn Claude, xem log,
    kill/rerun). Dùng cả hai = đủ tiêu chí.
